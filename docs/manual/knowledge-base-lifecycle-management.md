# 知识库生命周期管理（V4.0）

本文说明 V4.0 阶段的知识库文档生命周期能力：文档状态、软删除、重新建立索引，以及问答（Ask / RAG）如何过滤已删除文档与旧版本片段。

## V4.0 目标

在 V3.x 文档上传、异步摄入、事件时间线与处理分析的基础上，提供完整的**知识库文档生命周期管理**：

- 文档可标记为**已启用**或**已删除**；
- 支持**软删除**，删除后不再参与问答，但不立即物理清理数据；
- 支持**重新建立索引**，复用已保存的原始文本重建片段与向量；
- 问答只使用**当前有效索引版本**的文档片段。

## 文档状态说明

| 状态 | 中文展示 | 含义 |
|------|----------|------|
| `ACTIVE` | 已启用 | 文档处于启用状态，索引就绪后可参与知识库问答 |
| `DELETED` | 已删除 | 文档已软删除，不会再用于问答 |

此外，文档还有**处理状态**（如已上传、已切分、索引就绪、处理失败），用于判断索引是否可用于问答。

## 删除文档的行为

### 操作入口

- 页面：[文档管理](/documents.html) → 文档列表 → **删除文档**
- API：`DELETE /documents/{documentId}`

### 实际效果

- **软删除**：仅将 `lifecycle_status` 设为 `DELETED`，记录 `deleted_at`；
- **不物理删除** document 行、chunk 行、VectorStore 中的向量；
- **已删除文档不会再用于 Ask / RAG**：检索链路在应用层过滤已删除文档；
- 历史摄入任务、事件时间线仍可查看，便于审计与排障。

### 用户可见提示

删除前确认文案说明：删除后不会再用于问答，历史记录与底层索引不会立即物理清理。

## 重新建立索引的行为

### 操作入口

- 页面：[文档管理](/documents.html) → **重新建立索引**
- API：`POST /documents/{documentId}/reindex`（HTTP 202）

### 前置条件

- 文档必须为**已启用**；
- 必须存在可复用的**原始文本**（`source_text`），通常在上传时已保存；
- 若缺少原始文本，需重新上传文档（不支持在线编辑内容）。

### 处理流程

1. 创建 `REINDEX` 类型异步任务；
2. 基于 `source_text` 重新切分（chunking）；
3. 生成新 embeddings 并写入 VectorStore；
4. 成功后切换**当前索引版本**（`current_generation` +1），旧片段标记为**旧版本（SUPERSEDED）**；
5. 旧向量**不物理删除**，但应用层过滤，不再进入问答。

### 失败时的安全策略

若重新索引失败：

- 任务标记为失败，并记录事件与错误信息；
- **保留旧索引继续用于问答**（切换当前版本前不会废弃旧片段）；
- 失败的暂存片段会被清理，不影响当前可用索引。

### 用户可见提示

- 提交成功：「已提交重新索引任务，请在处理记录中查看进度。」
- 完成：「重新索引完成，新的文档片段已可用于知识库问答。」
- 失败：「重新索引失败，系统将保留旧索引继续用于问答。」

## Ask 过滤规则

知识库问答（`/rag/answers`）在检索各阶段统一过滤：

1. **只使用已启用文档**（`lifecycle_status = ACTIVE`）；
2. **只使用当前索引版本的 ACTIVE 片段**（`chunk.generation = document.current_generation`）；
3. 已删除文档、旧版本（SUPERSEDED）片段**不会进入**：
   - dense 检索结果；
   - lexical / hybrid 融合结果；
   - rerank 后最终结果；
   - citations 与 prompt context。

即使 VectorStore 仍返回旧向量，也会在应用层被排除。

## 事件时间线

在 [文档管理](/documents.html) 点击**查看处理记录**，可查看任务时间线，包括：

- 文档摄入各步骤；
- **重新索引**相关事件（请求、入队、开始、完成、失败）；
- 每步耗时（`durationMs`）与错误码、追踪 ID。

主展示为中文 `displayMessage`；技术详情可展开查看事件类型等。

## 当前明确不做

- 物理删除 document / chunks / VectorStore vectors；
- 向量库 delete-by-id；
- 文档恢复（undelete）；
- 版本 diff / 历史版本对比 UI；
- 文档内容在线编辑；
- 批量删除 / 批量重新索引；
- workspace / collection 隔离；
- 权限控制。

## 常见问题

### 删除后为什么还能看到历史记录？

删除是**软删除**。系统保留文档元数据、片段与向量，便于审计、排障与后续扩展；只是**不再参与问答**。

### 删除后为什么不立即物理清理向量？

为避免误删不可恢复、减少与向量库强耦合，V4.0 采用**应用层过滤**。旧向量可能仍占用存储，但不会被问答引用。

### 重新索引失败后还能问答吗？

可以。失败时系统**保留上一版有效索引**，旧片段继续标记为当前版本，直至新索引完整成功后才切换。

### 为什么旧片段不会再进入引用？

重新索引成功后，旧片段标记为 SUPERSEDED（旧版本），`current_generation` 指向新代。问答只匹配当前代的 ACTIVE 片段，因此引用中不会出现旧版本内容。
