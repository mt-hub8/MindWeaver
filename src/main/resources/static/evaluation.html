Evaluation Reports

系统支持对 RAG 检索质量进行离线评估，并对比 dense-only、rerank、hybrid 等策略差异。 本页说明指标含义与报告位置；不提供在线运行或图表 dashboard。

核心指标

HitRate@K（命中率@K）

TopK 结果中是否至少命中一条期望证据的 case 比例。越高说明「能找对」的问答越多。

Recall@K（召回率@K）

期望证据中被检索命中的比例。关注「该找的是否都找到了」。

Precision@K（精确率@K）

返回结果中有多少是期望相关的（本项目中为 matched / retrieved count）。

MRR / RR@K

平均倒数排名：第一个正确结果越靠前，分数越高。RR@K 为单 case 的 reciprocal rank。

对比模式

  • Dense baseline：仅向量检索，作为基线。
  • Dense + Rerank：扩大候选后 lexical 重排（V2.9）。
  • Hybrid：dense + lexical + RRF 融合（V3.0），可选再 rerank。

对比报告展示 baseline / variant 的 summary metrics 与 delta（improved / regressed / unchanged cases）。

报告输出路径

评估启用后，报告写入项目目录（默认）：

docs/evaluation/reports/

rag-retrieval-evaluation-report.json
rag-retrieval-evaluation-report.md
rag-retrieval-comparison-report.json      # dense vs rerank
rag-retrieval-comparison-report.md
rag-hybrid-comparison-report.json         # dense vs hybrid
rag-hybrid-comparison-report.md

若尚未运行评估，上述文件可能不存在。请按下方步骤手工 opt-in 运行。

如何手工运行 Evaluation

  1. 准备评估数据集：docs/evaluation/rag-retrieval-eval-cases.json
  2. 在 application.properties 中设置： 
    app.evaluation.retrieval.enabled=true
app.evaluation.retrieval.compare-hybrid=true   # 或 compare-rerank=true
  3. 启动应用，Runner 会在启动时执行评估并写入 docs/evaluation/reports/
  4. 阅读 JSON（机器可读）或 Markdown（人类可读）报告

详细说明见 docs/manual/rag-retrieval-evaluation.md、docs/manual/rag-two-stage-retrieval-rerank.md、docs/manual/rag-hybrid-retrieval-fusion.md。

注意事项

  • 默认 .\mvnw.cmd test 不运行真实 Qdrant / worker 评估。
  • 本页面不触发评估执行，仅作说明与入口指引。
  • Hybrid 对比不保证指标一定提升，报告如实记录 delta。
