5 分钟体验路径

  1. 启动服务（见 README「本地快速启动」）。
  2. 通过 API 上传文档并生成 embedding（POST /documents → POST /documents/{id}/embeddings）。
  3. 打开 Ask 页面提问，查看 citations 与 retrieval metadata。
  4. 在 Documents 确认文档与 chunks 已入库。
  5. （可选）按 Evaluation 说明运行检索评估并查看报告。

System Capabilities

  • RAG Answer API with Citations 回答附带引用片段，帮助用户追踪来源、降低幻觉风险。
  • Embedding Cache / Dedup 相同 chunk 不重复计算 embedding，节省成本与延迟。
  • Embedding Cache Metrics 查看缓存命中情况，便于观察去重效果。
  • Qdrant Manual Verification 可切换 Qdrant 向量库做手工验证（非默认测试路径）。
  • Retrieval Evaluation 用 HitRate、Recall、Precision、MRR 等指标判断检索质量，而不是凭感觉。
  • Two-stage Retrieval & Rerank 先扩大候选集再重排，提升关键词相关 chunk 的最终排名。
  • Hybrid Retrieval Fusion 同时兼顾语义相似（dense）与关键词精确匹配（lexical），经 RRF 融合。
  • Unified API Error Response 统一 code / message / traceId，便于前端与运维排查。
